# Etude de modification de l’immeuble pour réduire la consommation électrique

## Description du projet
### Semestre 1
Ce projet vise à intégrer et analyser les données issues de l’ADEME (Agence de la transition écologique) dans un data warehouse Postgre SQL à l’aide d’un pipeline ETL construit sur KNIME.
L’objectif principal est de centraliser, nettoyer et transformer les données afin de réaliser des analyses statistiques permettant d’identifier les facteurs influençant les évolutions environnementales et énergétiques.
### Semestre 2
Résoudre des besoins en Data Mining
##  Objectifs principaux

### Extract

Extraction des données brutes depuis les sources de l’ADEME (formats CSV).

### Transform

Transformation et nettoyage des données via KNIME :
- gestion des valeurs manquantes.
- harmonisation des unités et formats.
- traiter les valeurs incohérentes et abérrantes.
- création de variables dérivées.

### Load

Chargement (Load) des données dans un data warehouse Oracle structuré en schéma en étoile.

### Analyse

Analyse statistique et exploration des facteurs de modification à partir des données intégrées.

## Résultats attendus

- Une base de données intégrée et fiable dans Oracle.
- Une meilleure compréhension des facteurs de modification énergétique et environnementale.
- Des indicateurs prêts pour la visualisation et le reporting décisionnel via PowerBI.

## Technologies utilisées

- KNIME Analytics Platform
- Oracle Database
- SQL
- Python (en complément pour les statistiques avancées)
- PowerBI
